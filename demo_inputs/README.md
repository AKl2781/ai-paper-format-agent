# Demo Inputs

This directory is reserved for interview/demo input files.

Recommended files to place here:

- `messy_paper_sample.docx`
  - A de-identified paper/report sample that needs formatting cleanup.
  - Suggested issues: inconsistent headings, irregular body indentation or line spacing, reference/citation checkpoints, and figure/table numbering checkpoints.
- `template_sample.docx`
  - A de-identified formatting template sample.
  - It can be used to demonstrate optional template upload and fallback boundaries.

Current note:

- This README does not mean the real DOCX samples already exist in the repository.
- Do not claim this repository includes real demo papers unless actual `.docx` files are added later.
- Future work can place de-identified real paper/template samples in this directory for stable interview demos.

Suggested usage:

1. Put the paper sample at `demo_inputs/messy_paper_sample.docx`.
2. Put the template sample at `demo_inputs/template_sample.docx`.
3. Upload them through the existing frontend or API without changing core business logic.
