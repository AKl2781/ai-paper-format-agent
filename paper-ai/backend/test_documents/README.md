# Real Paper Test Corpus

本目录用于保存真实论文回归测试资产，目标是验证 Agent 面对真实 DOCX 时的稳定性。

注意：真实论文、学生论文、模板原件和任何包含个人信息的 DOCX 默认不提交到 git。提交前必须完成脱敏，并确认文件不包含姓名、学号、电话、邮箱、导师、学校内部编号等敏感信息。

## 目录说明

```text
test_documents/
  manifest.csv
  clean/
  messy/
  references/
  figures_tables/
  template_mismatch/
  reports/
  templates/
```

### clean

放置结构清晰、格式接近标准论文、参考文献和图表编号基本正确的 DOCX。

用于确认：

- 文档分类稳定。
- local 模式可以正常完成格式处理。
- 报告、预览、下载链路稳定。

### messy

放置格式较乱但仍属于论文的 DOCX，例如标题正文混排、段落缩进混乱、空段落较多、字体字号不统一、模板残留明显的文档。

用于确认：

- Agent 不会因为格式混乱而中断。
- 修改报告能给出人工复查建议。
- 在线预览不会出现大面积异常空白。

### references

放置参考文献相关问题样本，例如缺少参考文献章节、编号跳号、重复编号、正文引用不存在、文末文献未引用等。

用于确认：

- `reference_check` 字段稳定生成。
- 参考文献问题能进入人工复查项。

### figures_tables

放置图表编号相关问题样本，例如图编号跳号、表编号跳号、重复图号、重复表号、正文引用图表不存在、Word 表格没有表题等。

用于确认：

- `figure_table_check` 字段稳定生成。
- 图表编号问题能进入人工复查项。

### template_mismatch

放置论文与模板明显不匹配的样本，以及用于测试的脱敏模板。

用于确认：

- 模板异常不会导致主流程中断。
- 模板缺失字段时能够 fallback。
- 不会污染下载、预览、报告流程。

### reports

放置实验报告、课程报告、开题报告、综述报告等非标准论文或弱论文结构文档。

用于确认：

- 文档分类不会把所有 DOCX 都误判为标准论文。
- 需要用户确认的文档能进入确认流程。

### templates

放置脱敏后的学校或课程模板样本。

用于确认：

- 模板上传场景稳定。
- 模板缺失或异常时不阻断主流程。

## 新增测试文档规则

1. 先脱敏，再放入目录。
2. 文件名使用稳定编号，例如 `clean_001.docx`、`references_gap_001.docx`。
3. 每新增一篇文档，都要在 `manifest.csv` 记录预期结果。
4. 不要提交未经脱敏的真实 DOCX。
5. 不要把 `uploads/`、`outputs/`、回归生成结果复制到本目录。
