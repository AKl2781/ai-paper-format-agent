# v0.4.4 Composite Numbering Report

## Scope

本轮仅增强图表编号解析规则，支持复合编号作为独立编号参与去重和引用匹配。

未修改：

* formatter
* 上传接口
* 下载接口
* preview
* Agent 主流程
* scoring
* Risk Level 规则
* 真实 DOCX 样本

## Implemented

新增支持：

* `图2.1`
* `图2.2`
* `图2-1`
* `图2-2`
* `表3.1`
* `表3.2`
* `表A-1`
* `表B-2`

编号规范化策略：

* 纯数字编号仍保持为 `int`，例如 `图1` -> `1`。
* 复合编号使用字符串，点号和短横线统一为点号，例如 `表2-1` -> `2.1`。
* 字母章节编号统一为大写，例如 `表A-1` -> `A.1`。

## Result Analysis

复合编号支持后，章节式编号导致的重复误报有所下降。

主要改善：

* `表2-1`、`表2-2`、`表2-3` 不再统一截断为 `表2`。
* `表3.1`、`表3.2`、`表3.3` 不再统一截断为 `表3`。
* `图1.1`、`图1.2`、`图1.3` 不再统一截断为 `图1`。

仍然存在 high_risk：

* 中英文双语题名可能仍被识别为重复编号，例如中文表题和 `Table 1` 同时出现。
* 模板说明文字中的引用示例仍可能被当成正文引用。
* 参考文献格式示例反复从 `[1]` 开始，仍可能触发重复参考文献编号。
* 部分模板中的图表引用和实际图表题名仍存在上下文误判。

## Conclusion

v0.4.4 达成目标：复合图表编号不再被统一截断为首位数字，真实回归保持 16/16 PASS。

本轮没有改变 Risk Level 规则，因此剩余 high_risk 不在本版本继续降级。后续应进入模板上下文识别和双语题名去重，而不是继续扩大编号解析规则。
